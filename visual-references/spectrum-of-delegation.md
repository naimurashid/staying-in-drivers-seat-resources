# The spectrum of delegation

Three categories of work, by how much AI should do.

| Usually safe to delegate | Delegate with review | Stay hands-on |
|---|---|---|
| Boilerplate & scaffolding | Analysis code | Architecture & design |
| Syntax & idioms | Database queries | Methodological choices |
| Format conversions | Methods-section drafts | Core algorithms |
| Search-query brainstorming | Prose smoothing | Interpretation & defense |
|  | Literature summaries | Diagnosing the room |
|  | Worked examples & rubrics |  |

## Reading the table

**The columns are stable.** The categories - what's safe, what needs review, what stays hands-on - don't move.

**The items shift.** Which items go in which column changes by activity, by stakes, by your career stage. A new researcher's "stay hands-on" column has more items than a senior researcher's. A regulated-trial statistician's "delegate with review" column has more items than a pure-academic statistician's.

## Three prompts to pull AI out of executor mode

- *"Ask me three questions before suggesting a method."*
- *"Don't give code yet - ask me to specify the estimand."*
- *"Argue against this model choice and list two alternatives."*

See [`tutor-critic-prompts.md`](../prompts/tutor-critic-prompts.md) for more starter prompts.

## The line is personal

Where the columns sit on this grid is yours, not someone else's. It will be different for you than for a colleague. Different for your trainees than for either of you. Different for code than for analysis than for teaching.

**The discipline isn't where you draw the line. It's that you are *deliberate* about drawing one at all.**
